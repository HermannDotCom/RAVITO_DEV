## Color Scheme

### Primary Colors
- **Orange (#ea580c)**: Active toggles, primary buttons, active tab
- **Gray (#6b7280)**: Inactive toggles, disabled states
- **Green (#10b981)**: Success messages
- **Red (#ef4444)**: Error messages
- **Blue (#3b82f6)**: Info messages

### Module Toggle Colors
- **Active Background**: `bg-orange-600`
- **Inactive Background**: `bg-gray-200`
- **Hover Border**: `border-orange-300`
- **Focus Ring**: `ring-orange-500`

### Card Styling
- **Background**: White (`bg-white`)
- **Border**: Gray 200 (`border-gray-200`)
- **Shadow**: Small shadow (`shadow-sm`)
- **Border Radius**: `rounded-lg` (0.5rem)

## Interactions

### Toggle Click Flow
1. User clicks toggle
2. Toggle immediately shows loading spinner
3. API call is made to update permission
4. If successful:
   - Toggle updates to new state
   - Green success toast appears for 2 seconds
   - Spinner disappears
5. If error:
   - Toggle reverts to previous state
   - Red error toast appears for 3 seconds
   - Spinner disappears

### Debouncing
- Changes are debounced with 500ms delay
- Multiple rapid clicks queue properly
- Only the final state is saved

## Accessibility Features

### ARIA Labels
- Toggle buttons have `role="switch"`
- Toggle buttons have `aria-checked` attribute
- Each toggle has descriptive `aria-label`

### Keyboard Navigation
- Tab navigation through all toggles
- Space/Enter to activate toggle
- Focus visible with orange ring

### Touch Targets
- Minimum 44px touch target size
- Adequate spacing between toggles (12px gap)
- Large clickable areas

## Responsive Breakpoints

- **Mobile**: < 640px - Single column layout
- **Tablet**: 640px - 1024px - Adapts grid
- **Desktop**: > 1024px - 2 column grid for toggles

## Animation Timings

- **Toggle Animation**: 200ms ease-out
- **Toast Slide In**: 300ms ease-out
- **Toast Duration**: 2s (success), 3s (error)
- **Hover Transition**: 150ms
- **Loading Spinner**: Continuous rotation

## Browser Support

Works on all modern browsers:
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

## Performance Considerations

- Permissions loaded once on mount
- Updates happen individually (not bulk reload)
- Map-based lookups for O(1) permission checks
- Minimal re-renders with proper React memoization
